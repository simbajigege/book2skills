# Skill: 资产质量检查（财报透视 — Module 2）

**知识来源**：《一本书读懂财报》肖星著，清华大学经管学院教授，2019年修订版
**案例库**：`references/case_library.md`（所有 CASE-M2-XXX 原文引用均来自此文件）
**独立使用**：本 skill 可单独调用，也可接收 M1 行业基准报告后作为编排流程第二步

## 调用时机

以下任一情况可单独调用本 skill：
- 用户想检查某公司资产负债表中的各资产项目是否存在风险
- 用户担心某公司存在资产虚增、关联方占款、或利润操纵
- 用户提供了应收账款、存货、固定资产、无形资产等数据，想要评估质量
- 作为完整财报分析的第二步（建议先运行 M1 行业背景分析建立行业基准）

**若未提供 M1 行业基准报告**：执行本 skill 时自行询问用户行业类型，
参考 `subskills/m1_industry_analysis/references/case_library.md` 中的行业类型参照表。

---

## 模块输入

来自 orchestrator 的传入信息：
- M1 行业坐标基准报告（行业类型、合理区间、上下游议价权初判）
- 用户提供的资产负债表数据（越详细越好）：
  - 货币资金、应收账款、其他应收款、预付账款、存货
  - 固定资产、无形资产、长期待摊费用
  - 最好有连续 3 年数据

## 模块输出

向 orchestrator 返回「资产质量报告」，包含：
- 各资产项目的健康评级（🔴风险 / 🟡关注 / 🟢正常）
- 发现的具体红旗信号及原文案例佐证
- 需要用户进一步核查的问题列表

---

## 执行流程

### 总览：资产质量检查的五个维度

```
资产负债表（左边）
├── 流动资产
│   ├── Step 1：其他应收款  ← 关联方占款风险
│   ├── Step 2：应收账款    ← 坏账与回收质量
│   ├── Step 3：存货        ← 真实性与毛利操纵
│   └── Step 4：预付账款    ← 异常大额预付
└── 非流动资产
    ├── Step 5：固定资产    ← 历史成本虚增风险
    └── Step 6：无形资产    ← 研发资本化操纵
```

每步执行：计算指标 → 与 M1 行业区间对比 → 触发红旗则引用对应案例 → 记录评级

### Step 1：其他应收款——关联方占款扫描

**核心问题**：这笔"其他人欠公司的钱"是正常业务往来，还是资金被关联方拿走了？

**计算指标**：
```
其他应收款占总资产比 = 其他应收款 / 总资产
其他应收款 vs 月营业额 = 其他应收款 / (年营收 / 12)
```

**判断规则**：

| 信号强度 | 条件 | 评级 |
|---------|------|------|
| 🟢 正常 | 占总资产 < 2%，且金额合理（备用金、押金规模） | 正常 |
| 🟡 关注 | 占总资产 2%-5%，或余额相当于超过 1 个月营收 | 需说明用途 |
| 🔴 红旗 | 占总资产 > 5%，或金额以亿元计，或长期挂账不减 | 关联方占款风险 |

**红旗触发时 → 引用 CASE-M2-002**，说明：
> 备用金和押金不可能有几十亿，大额其他应收款是关联方通过上市公司套取资金的惯用路径。

**追问用户**：
- 其他应收款的前三大往来方是谁？是否为关联方或大股东关联实体？
- 余额是近年新增的，还是多年挂账？有无还款记录？
- 年报附注中是否有说明这笔钱的用途和预计回收时间？

---

### Step 2：应收账款——回收质量与坏账风险

**核心问题**：账面上那些"客户欠的钱"，真的能收回来吗？

**计算指标**：
```
应收账款占营收比 = 应收账款 / 年营业收入
应收账款周转天数 = 365 / (年营收 / 应收账款)
坏账准备覆盖率 = 坏账准备 / 应收账款总额
```

**判断规则**：

| 信号强度 | 条件 | 评级 |
|---------|------|------|
| 🟢 正常 | 周转天数符合行业惯例；坏账准备计提充分 | 正常 |
| 🟡 关注 | 应收账款增速连续超过营收增速；周转天数拉长 | 赊销扩张，需核查账龄 |
| 🔴 红旗 | 2年以上账龄应收款占比 > 20%；坏账准备率远低于同行 | 坏账风险高，利润虚增 |

**对比 M1 行业基准**：
- to B 重资产行业（通信设备、工程）：周转天数 60-120 天属正常
- to C 消费品行业：周转天数 > 30 天需警惕
- 医药/医疗器械行业：医院回款慢，90-180 天是行业特性

**红旗触发时 → 引用 CASE-M2-004**，说明：
> 应收账款不是已赚到的钱，收不回来就是净利润的直接损失。
> 1300万应收账款中只有200万坏账，就能把净利润从520万砍到320万（降38%）。
> 大额应收账款长期挂账的风险，彩电企业22亿坏账一次性造成数十亿亏损就是前车之鉴。

**追问用户**：
- 应收账款的账龄分布如何？（1年内/1-2年/2年以上各占多少）
- 前五大应收款客户是否集中？有无单一大客户依赖？
- 坏账准备的计提方法和比例，与同行相比是否偏低？

### Step 3：存货——真实性与毛利操纵验证

**核心问题**：存货余额反映的是真实库存，还是被用来操纵毛利率的工具？

**计算指标**：
```
存货占总资产比 = 存货 / 总资产
存货周转天数 = 365 / (营业成本 / 存货)
存货增长率 vs 营收增长率（3年趋势）
毛利率 vs 存货余额（同向变动检验）
```

**判断规则**：

| 信号强度 | 条件 | 评级 |
|---------|------|------|
| 🟢 正常 | 存货周转天数符合行业均值；存货与营收同向增减 | 正常 |
| 🟡 关注 | 存货周转天数拉长；或存货增速持续高于营收增速 | 产品积压风险 |
| 🔴 红旗 | **存货增加 + 毛利率同期提升**（两者同向上行） | 固定成本操纵毛利润 |

**⚠️ 特别检验：存货-毛利率同向变动检验**

这是本书最独特的诊断方法，针对高固定成本的重资产行业：

```
检验逻辑：
若 存货余额↑ 且 毛利率↑  →  🔴 疑似固定成本操纵，毛利率提升不可信
若 存货余额↓ 且 毛利率↓  →  🟡 去库存压力，但毛利下降可能是正常的（去年成本今年摊）
若 存货余额↑ 且 毛利率↓  →  🟢 产品积压但未操纵，需关注去化节奏
若 存货余额↓ 且 毛利率↑  →  🟢 真实改善信号，货卖掉了，利润提升
```

**红旗触发时 → 引用 CASE-M2-003**，说明：
> 晖平公司生产了2000件，卖了1000件，毛利润反而比去年全部卖掉1000件时还高——
> 因为等量固定成本被摊到更多产品上，进了存货而非成本。
> 存货和毛利率同时上升，是"表面好看但实质是积压"的经典信号。

**对比 M1 行业基准**：
- 重资产制造业（固定成本高）：此信号尤其需要警惕
- 轻资产行业（变动成本主导）：此效应较弱，但仍需验证

**追问用户**：
- 存货的构成是什么？原材料/在产品/产成品各占多少？
- 产成品的账龄分布（库龄）？是否有滞销产品？
- 今年产量是否大幅超过销量？（如有生产数据）

---

### Step 4：预付账款——异常大额预付扫描

**核心问题**：预付账款是正常商业预付，还是变相转移资金的通道？

**计算指标**：
```
预付账款占总资产比 = 预付账款 / 总资产
预付账款 vs 采购规模 = 预付账款 / 年营业成本
```

**判断规则**：

| 信号强度 | 条件 | 评级 |
|---------|------|------|
| 🟢 正常 | 预付账款与采购节奏匹配；周期性波动 | 正常商业预付 |
| 🟡 关注 | 预付账款持续增大，但采购规模未同比增长 | 需核查预付对象 |
| 🔴 红旗 | 预付对象为关联方；预付后长期未见收货入账 | 资金转移风险 |

**说明**：
- 正常预付：供应商要求先付款（供应紧张、买方议价权弱），最终对应收到货物入库
- 异常预付：预付后迟迟不收货，或预付给关联方，实质是以"预付"之名转移资金
- 如 M1 判断该公司上游供应商议价权强，预付账款偏高属行业特性，不必过度告警

**追问用户**：
- 预付账款的前三大预付方是谁？是否为关联方？
- 预付账款最终是否对应了货物入库？账期是否正常？

### Step 5：固定资产——历史成本真实性验证

**核心问题**：固定资产的账面价值，是否对应真实发生的资本支出？

**计算指标**：
```
固定资产占总资产比 = 固定资产净值 / 总资产
固定资产净值 vs 历年资本支出累计（现金流量表）
单位产能固定资产值 = 固定资产 / 年产能（若有产能数据）
```

**判断规则**：

| 信号强度 | 条件 | 评级 |
|---------|------|------|
| 🟢 正常 | 固定资产占比符合 M1 行业基准；与历年资本支出匹配 | 正常 |
| 🟡 关注 | 固定资产金额远超行业同类产能的建造成本基准 | 需核查资产来源 |
| 🔴 红旗 | 某类资产（如生物资产、林地、矿权）账面额与合理成本差异巨大 | 虚增资产风险 |

**历史成本验证方法**：
1. 查行业内同等规模资产的建造/购置成本（公开数据、招股书等）
2. 与该公司历年资本支出（购建固定资产支付的现金）累计值对比
3. 政府划拨资产（土地、矿权）成本接近零，若账面大额需追问确权和入账依据

**红旗触发时 → 引用 CASE-M2-001**，说明：
> 造纸企业声称种树花了6亿，实际只花了3000万，5.7亿是虚增资产。
> 历史成本原则只允许记录真实花出去的钱。若账面远超合理建造成本，差距就是需要追问的数字。

**⚠️ 高风险资产类别**（更难独立核查）：
- 生物资产（林木、养殖）：成本真实性难以外部验证
- 在建工程：长期挂在在建工程不转固，可能是延缓折旧
- 矿产资源资产：探矿权估值主观性强
- 境外资产：监管触达难度大

**追问用户**：
- 固定资产中，哪类资产金额最大？其建造/购置历史如何？
- 是否有长期挂账的在建工程（超过正常建设周期）？
- 审计师对固定资产是否出具过保留意见或特别说明？

### Step 6：无形资产——研发资本化异常检测

**核心问题**：无形资产中的自研技术，是否违反会计准则被不当资本化？

**计算指标**：
```
无形资产占总资产比 = 无形资产 / 总资产
自研技术占无形资产比 = 自主研发无形资产 / 无形资产合计
研发费用率 = 研发支出（费用化） / 营业收入
```

**判断规则**：

| 信号强度 | 条件 | 评级 |
|---------|------|------|
| 🟢 正常 | 无形资产主要为外购技术、商标、土地使用权；研发费用率符合行业均值 | 正常 |
| 🟡 关注 | 自研技术占无形资产比例 > 30%；研发费用率明显低于同行 | 可能存在资本化操纵 |
| 🔴 红旗 | 亏损或微利公司 + 自研无形资产持续增长 + 研发费率远低于同行 | 研发资本化掩盖亏损 |

**会计准则核心规则**（供 agent 向用户解释）：
- 技术研究阶段支出 → 全部费用化，进管理费用
- 开发阶段支出满足一定条件 → 可资本化
- 申请专利费用 → 可资本化
- 外购技术 → 可资本化
- 通过独立法人研发子公司"外购"自研技术 → 规则上可资本化，但实质是绕过规定

**红旗触发时 → 引用 CASE-M2-005**，说明：
> 花50万研发技术，只有5000元申请专利的部分可以进无形资产，其余只能走费用。
> 但亏损企业为让报表好看，倾向于将研发支出资本化——直接效果是减少当期亏损。
> 若公司恰好处于亏损边缘，且无形资产中自研部分持续增长，需高度警惕。

**追问用户**：
- 无形资产附注中，各类别（土地使用权/商标/技术/软件）各占多少？
- 公司是否设有独立的研发子公司，且向上市主体出售技术？
- 研发费用率与同行比较如何？（同行数据可从竞争对手年报获取）

## 资产质量诊断的核心逻辑图

```
账面资产额
    ↓
① 这笔资产对应真实的钱吗？（历史成本验证）
    → 否 → 🔴 固定资产虚增（CASE-M2-001）
    ↓
② 这笔资产是公司的还是被别人借走了？（其他应收款）
    → 被借走 → 🔴 关联方占款（CASE-M2-002）
    ↓
③ 这笔资产最终能变回现金吗？（应收账款/存货质量）
    → 收不回 → 🔴 坏账风险（CASE-M2-004）
    → 卖不掉 → 🟡 存货积压
    ↓
④ 毛利率与存货是否有矛盾信号？（CASE-M2-003）
    → 存货↑ 且 毛利率↑ → 🔴 固定成本操纵
    ↓
⑤ 亏损是否被资产化掩盖？（无形资产研发资本化）
    → 是 → 🔴 研发资本化操纵（CASE-M2-005）
    ↓
输出资产质量报告，传回 orchestrator
```

---

## 不应做的事

- ❌ 不能孤立评价某个资产项目，必须结合 M1 的行业基准
- ❌ 不能把"其他应收款大"等同于"一定有问题"，要结合规模和用途
- ❌ 不能只报告红旗，必须说明依据和追问方向，帮用户知道下一步查什么
- ❌ 数据不足时不要猜测，明确列出"缺少XXX数据，该项无法评级"
- ❌ 不要给出"买入/卖出"建议，只提供分析框架和风险信号识别
