# Skill: Personalization (Creating Superfans - Module 4)

**Knowledge source:** *Creating Superfans* by Brittany Hodak.
**Reference library:** `references/case_library.md`
**Standalone use:** This module can run independently, but works best after M3 customer understanding.

## When to Use

Use this module when the user asks how to personalize a message, customer journey, gift, follow-up, onboarding moment, CRM trigger, loyalty action, or service experience.

## Module Inputs

- Customer segment and journey stage
- Known customer preferences, behaviors, dates, milestones, or context
- Available CRM or support data
- Channel constraints and privacy boundaries
- Desired customer feeling or action

## Module Outputs

Return a personalization plan with:
- Personalization level
- Customer signal used
- Specific action or message
- Repeatable trigger
- Boundary or privacy warning

## Execution Flow

### Overview

```text
Identify customer signal -> choose personalization level -> design meaningful action -> make it repeatable without becoming intrusive
```

### Step 1: Choose the personalization level

**Core question:** Is the brand treating customers as people, not transactions?

**Decision rules:**

| Level | Example |
|-------|---------|
| Basic | Name, account, purchase, lifecycle stage |
| Contextual | Timing, use case, behavior, journey moment |
| Story-based | Goals, family, identity, values, milestones |
| Scaled | CRM trigger that still reflects real customer context |

**Ask the user:** What specific customer detail can be used respectfully and helpfully?

**Citation instruction:** When explaining personalization expectations, cite `#personalization-expectations` or `#personalization-at-scale`.

### Step 2: Apply the Platinum Rule

**Core question:** Are we treating customers the way they want to be treated?

**Decision rules:**

| Strong personalization | Weak personalization |
|------------------------|----------------------|
| Reflects customer preference or story | Uses only first-name merge fields |
| Helps, recognizes, or delights the customer | Shows off brand cleverness |
| Feels human and appropriate | Feels intrusive or manipulative |

**Ask the user:** Would the customer see this as useful attention or uncomfortable surveillance?

**Citation instruction:** When using the treatment principle, cite `#platinum-rule`.

### Step 3: Design the gesture or trigger

**Core question:** What small action would make this customer feel remembered?

**Decision rules:**

| Situation | Recommendation |
|-----------|----------------|
| Gift | Make it unique and about the customer, not logo-heavy |
| Milestone | Mark the date in a way that recognizes the customer's story |
| Follow-up | Mention the actual context or goal the customer shared |
| CRM workflow | Trigger a human or semi-human action at a meaningful moment |

**Ask the user:** What date, person, interest, or milestone matters to this customer?

**Citation instruction:** For gifts or gestures, cite `#gifting-principles`, `#gifts-not-ads`, `#make-it-unique`, or `#small-personal-touches`.

## Do Not

- Do not call first-name merge fields meaningful personalization.
- Do not recommend logo-heavy gifts that function as ads.
- Do not use customer data in a way that feels creepy.
- Do not make personalization expensive when attention would work better.
